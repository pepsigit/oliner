Rigidbody

刚体

  • Mass : 质量

    • 质量越大，惯性越大。官方建议物体质量之间最好不要相差100倍。防止2个物体碰撞后，产生过大的速度，影响性能。

  • Drag : 阻力

    • 指的是空气阻力。物体受到这个力的作用，数值越大，移动越困难。设置为无限时，物理受到外力，立即移动。

  • Angular Drag : 角阻力

    • 指的是空气阻力。物体受到这个力的作用，数值越大，旋转越困难。设置为无限时，物理受到外力，立即旋转。

  • Use Gravity : 重力是否启用

    • 启用：物体受到重力影响。

  • Is Kinematic : 运动学是否启动

    • 启用 : 运动学刚体不受力、碰撞或关节控制，完全由动画或代码Transform驱动，但运动学刚体可以通过碰撞或关节影响其他刚体的运动。非运动学刚体则受力、碰撞或关节控制。

  • Interplate : 差值计算类型

    • None : 不使用差值

    • Interpolate : 根据上一帧来平滑移动

    • Extrapolate : 推算下一帧物体的位置来平滑移动

  • Collision Detection : 碰撞检测方式

    • Discrete（离散）: 默认的碰撞检测方式。若当物体A运动很快的时候，有可能前一帧还在B物体的前面，后一帧就在B物体后面了，这种情况下不会触发碰撞事件，所以如果需要检测这种情况，那就必须使用后两种检测方式。

    • Continuous（连续）: 这种方式可以与有静态网格碰撞器的游戏对象进行碰撞检测。

    • Continuous Dynamic（动态连续）: 这种方式可以与所有设置了Continuous或Continuous Dynamic方式的游戏对象进行碰撞检测。

  • Freeze Position/Rotation（冻结位置/旋转）:

    • 可以对物体在X、Y、Z三个轴上的位置/旋转进行锁定，即使受到相应的力也不会改变，但可以通过脚本来修改。

Tips:

  • 在使用刚体来控制物体运动时，可以通过添加力的方式来操作对象，这与直接操作Transform相比，视觉上有很大的不同。

  • 同时以物理方式和Tranform方式操作物体，会导致碰撞及其相关运算问题。

  • 父子级 :

    • 父(刚)子(无) :

      • 移动父物体，父子会做物理运动，重心为两者组成的包围盒重心。

      • 移动子物体，会更新重心，拉动父物体，并做物理运动。

    • 父(无)子(刚) :

      • 移动父物体，子物体跟着移动，但是子物体是半独立的，自身会受到物理影响。

      • 移动子物体，子物体自身会做物理运动，对父物体无影响。

    • 父(刚)子(刚) :

      • 移动父物体，子物体跟着移动，但自身会受到物理影响。

      • 移动子物体，子物体自身会做物理运动，对父物体无影响。

  • 脚本控制 :

    • 使用物理引擎控制刚体时，不要直接操作Transform。

  • 动画 :

    • 有时候需要纸娃娃效果，需要在动画与物理控制之间进行切换。当我们勾选Is Kinematic后，刚体为运动学刚体，它将不再受到外力影响。这时只能通过Transform方式来操作对象，但是运动学刚体还会影响其他刚体，但他自己不会再受物理引擎控制。


Character Controller

角色控制器

  • Slope Limit : 坡度限制

    • 只能攀爬<=此参数值的坡度

  • Step Offset : 台阶高度

    • 可以迈上的最高的台阶高度

  • Skin Width : 皮肤厚度

    • 参数决定了两个碰撞体可以相互参入的深度，较大的参数值会产生抖动的现象，较小的参数值会导致所控制的游戏对象被卡住，较为合理的设定是，该参数值为radius值得10%

  • Min Move Distance : 最小移动距离

    • 如果所控制的角色对象移动的距离小于该值，则游戏对象将不会移动这可以避免抖动，大多数情况下将该值设为0

  • Center : 碰撞体中心

    • 碰撞体在世界坐标中的位置，并不影响所控制的角色对象的中心坐标

  • Radius : 半径

    • 胶囊体碰撞的长度半径，同时该项也决定了碰撞体的宽度

  • Height : 高度

    • 胶囊体碰撞体的高度，改变此值将会使碰撞体沿着Y轴的正负连个方向的伸缩

  • 函数

    • CollisionFlags Move () : 移动

      • param :

        • motion ( Vector3 ) : 移动方向增量

      • return :

        • CollisionFlags : 碰撞标识

    • bool SimpleMove () : 移动

      • param :

        • motion ( Vector3 ) : 移动方向增量

      • return :

        • bool : 是否接触地面

    • void OnControllerColliderHit () : 碰撞检测

      • param :

        • hit ( ControllerColliderHit )[非自定义传值] : 碰撞信息

  • Move 和 SimpleMove 的区别

    • Move :

      • 角色移动只受到碰撞的约束

      • 角色遇到碰撞时将会沿着碰撞盒滑动

      • 返回值是对移动过程中遇到的碰撞信息的汇总

      • 函数并不使用重力

    • SimpleMove :

      • Y轴的速度将会被忽略

      • 速度单位是米每秒

      • 重力自动生效

      • 返回值是角色是否着地


PhysicsManager

物理管理器

  • Gravity : 重力

    • 重力应用于所有刚体，一般重力仅在Y轴起作用（y轴负方向是向下）。单位是 m/s2

  • Default Material : 默认材质

    • 如果一个碰撞体没有设置物理材质的话，将会设置为默认材质

  • Bounce Threshold : 反弹阈值

    • 如果2个碰撞体的相对速度低于该值，那么将不会反弹。该值也用于减少抖动，所以不建议设置为很低的值

  • Sleep Threshold : 静止阈值

    • 速度低于此值，意为静止

  • Default Contact Offset : 默认碰撞检测距离

  • Solver Iteration Count : 求解迭代次数

    • 决定了关节和链接的计算精度。一般设置为7就可以适用于大多数情况

  • Queries Hit Triggers : 命中触发

  • Enable Adaptive Force : 自适应力

  • Layer Collision Matrix : 层之间碰撞检测矩阵
